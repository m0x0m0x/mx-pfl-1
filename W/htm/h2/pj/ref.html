Why PicoCSS is classless?

Because semantic HTML should be enough. Pico targets raw HTML elements — <article>, <section>, <nav>, etc. It also provides extra utilities if you need them, but its core philosophy is no classes required for beautiful design.

Responsive by default

Built with modern CSS (flex, grid, media queries) and respects user preferences for dark/light mode.

🔲 Responsive grid (Pico utility `.grid`)

While Pico is classless by default, it includes a flexible grid system using .grid to create columns that wrap on mobile. It's optional but extremely handy.

Column 1

With .grid on a parent, children become equal-width columns. On narrow screens, they stack vertically.

🏗️ Container and spacing

By default, the <body> has max-width and auto margins, ensuring content is centered and readable. You can also wrap sections in <div> without extra hassle. 

<!-- No classes required for default styling — but Pico provides helpers -->
<article>
  <header>Title</header>
  <p>Content</p>
</article>

<!-- .grid for responsive columns -->
<div class="grid">
  <div>Column 1</div>
  <div>Column 2</div>
</div>

🎨 Buttons and variants

Pico gives you default, .secondary, .contrast, and outline variants via role="button" or classes on buttons.

Link as button

Note: Pico uses .outline for bordered transparent buttons, and role="button" on anchors to style them consistently.

⚡ Responsive breakpoints & variables

PicoCSS sets global CSS variables (--pico-font-size, --pico-primary) which you can override. But out of the box, everything is responsive — try resizing the browser.

🌗 Dark/Light mode: automatically follows system settings. No JavaScript, no toggle needed. Pico's philosophy: respect user preference.

📱 Mobile ready: typography scales, form inputs fill width, grid columns wrap.
