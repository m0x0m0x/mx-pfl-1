📦 Core Features

🔐 JWT Auth POST /auth/login

Secure token-based authentication with access and refresh tokens. Supports role-based permissions and token blacklisting.

  • Access token (15min)
  • Refresh token (7 days)
  • Role-based guards
Docs Test

👥 User Management /api/users

Full CRUD operations for user resources with validation, pagination, and filtering. Connected to PostgreSQL via SQLAlchemy.

  • GET, POST, PUT, DELETE
  • Query parameters filtering
  • Pagination support
Schema Endpoints

🗄️ PostgreSQL + SQLAlchemy

Async-ready database layer with migration support (Alembic), connection pooling, and model relationships.

  • SQLAlchemy 2.0 ORM
  • Alembic migrations
  • Asyncpg support
Models Migrations

📋 Request Validation Marshmallow

Automatic request/response serialization with detailed error messages. OpenAPI spec generated from schemas.

  • Input validation
  • Swagger UI ready
  • Custom error formats
Schemas Swagger

🧪 Pytest Coverage

Comprehensive test suite with fixtures, factories, and isolated test database. 90%+ code coverage maintained.

  • Pytest + factory_boy
  • Test isolation
  • CI/CD ready
Tests Coverage

🚀 Docker & Deploy

Containerized with Docker Compose. Ready for Render, Railway, or any cloud platform. Gunicorn + Uvicorn workers.

  • Dockerfile + compose
  • Environment configs
  • Health check endpoint
Docker Deploy

💡 Why This Flask API?

This API was built with simplicity and developer experience in mind. Every endpoint follows RESTful conventions and returns consistent JSON responses. The project structure is modular, making it easy to extend with new resources without touching existing code.

🛠️ Tech Stack

Python 3.12 Flask 3.0 PostgreSQL 16 SQLAlchemy 2.0 JWT Docker Pytest

Get started in minutes. Clone the repository, set your environment variables, and run docker compose up. The API comes with a built-in Swagger UI at /docs for easy exploration and testing.
